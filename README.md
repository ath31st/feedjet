## Описание

Feedjet — fullstack-приложение для организации показа контента на информационных экранах.

- **Киоск** — полноэкранный плеер сценария по адресу `/` или `/:slug` (есть режим превью `/preview/:slug`).
- **Панель администратора** — панель управления по адресу `/admin` (доступ по JWT).
- Обновления на киоске в реальном времени через **SSE**.
- Управление питанием и перезагрузкой экранов через интеграции с устройствами.

## Возможности

### Киоск

- Сценарии показа: виджеты (`расписание`, `RSS-ленты`, `дни рождения`, `инфо`), изображения и видео с заданной длительностью и порядком
- Бегущая строка
- Темы оформления, поворот экрана, режим анимации, сезонный оверлей
- Получение обновлений конфигурации, ленты, брендинга и команд управления по SSE

### Панель администратора

- Сценарии и медиатека (загрузка изображений/видео, папки, превью)
- RSS-ленты и параметры карусели
- Расписание мероприятий
- Дни рождения (ручной ввод и импорт из файлов), фоны по месяцам
- Оформление и брендинг (логотип, тема)
- Бегущая строка
- Киоски и график работы экрана (включение/выключение по расписанию)
- Устройства и интеграции: Fully Kiosk, ADB, Philips JointSpace
- Просмотр и выгрузка логов сервера

### Сервер

- API на tRPC, аутентификация JWT
- SQLite + Drizzle ORM, миграции
- Фоновые задачи через `node-cron` (RSS, кеш изображений, опрос состояния экранов, очистка устройств и т.д.)
- Обработка медиа: Sharp (изображения), FFmpeg (превью и длительность видео)
- Погода через OpenWeatherMap (отключается вместе с RSS при `OFFLINE_MODE=true`)

## Архитектура

Клиент и сервер связаны типобезопасным контрактом tRPC.

### Клиент (Feature-Sliced Design)

| Слой | Назначение |
|------|------------|
| `app` | Инициализация, layouts, providers, маршруты |
| `pages` | Киоск, админка, логин, super-admin, 401/403/404 |
| `widgets` | Крупные UI-блоки админки и киоска |
| `features` | Пользовательские сценарии (auth, сценарии, RSS, устройства и др.) |
| `entities` | Доменные модели и API-хуки |
| `shared` | UI-kit, утилиты, стили, API-клиент |

### Сервер (слоистая структура)

| Каталог | Назначение |
|---------|------------|
| `config` | Конфигурация и переменные окружения |
| `trpc` | Роутеры API |
| `services` | Бизнес-логика |
| `db` | Схема и миграции Drizzle |
| `integration` | Fully Kiosk, ADB, Philips JointSpace, OpenWeather |
| `cron` | Периодические задачи |
| `sse` | Server-Sent Events |
| `middleware` | Auth и прочее |
| `validations` | Zod-схемы |
| `mappers`, `errors`, `utils`, `types` | Вспомогательные слои |

Отдельного воркер-процесса нет: API, cron и SSE работают в одном процессе Node.js.

Основные HTTP-пути: `/trpc`, `/sse/stream/:kioskId`, статика `/videos`, `/images`, `/logos`, `/backgrounds`, `/cache`.

## Локальный запуск

Требования: Node.js 20+, Yarn Classic v1, FFmpeg, build-essential (для native-модулей).

```bash
yarn install

cp server/.env.example server/.env
cp client/.env.example client/.env
# заполните обязательные переменные (см. комментарии в .env.example)

cd server && yarn db:migrate
cd server && yarn dev    # API, по умолчанию :3000
cd client && yarn dev    # Vite, по умолчанию :5173
```

Опционально HTTPS через Nginx:

```bash
node scripts/setup-nginx.js
sudo nginx -c /path/to/feedjet/nginx/nginx.conf
```

Сборка:

```bash
cd server && yarn build && yarn start
cd client && yarn build
```
